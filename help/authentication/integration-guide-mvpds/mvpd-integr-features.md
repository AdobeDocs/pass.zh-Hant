# MVPD整合功能

>[!NOTE]
>
>此頁面上的內容僅供參考。 使用此API需要Adobe的目前授權。 不允許未經授權的使用。

## 概觀 {#mvpd-int-features-overview}

Adobe Pass驗證支援各處電視的新興標準。 它符合&#x200B;**CableLabs OLCA （線上內容存取）規格**，此規格提供從線上來源傳送視訊給付費電視客戶的技術需求與架構。 Adobe於2011年6月參與了CableLabs的Interopt聯合測試專案，並通過了服務供應商實作的測試程式。  Adobe也是&#x200B;**OATC （開放驗證技術聯盟）**&#x200B;的現用成員，並參與幾個子委員會的規格起草專案，作為該主體的一部分。

在說明Adobe Pass驗證OLCA法規遵循，以及Adobe參與OATC的情況後，請務必注意Adobe Pass驗證實際上是「不受通訊協定限制」的。  但在TVE時代的這個階段，Adobe Pass驗證無疑是以OLCA標準為導向。  這些標準說明不同TVE播放器（程式設計人員、MVPD和服務提供者）實施TVE功能的目前議定方法。 下表列出許多這類功能，並提供相關頁面的連結，其中提供詳細資訊和如何實作這些功能的範例。

表格中的資訊旨在推動Adobe Pass驗證整合程式，以在所有MVPD整合中達成一致功能。 優先順序欄將功能分為A、B和C：

* 答：這些是整合初始推出的「必須具備」功能。
* B — 這些是初始整合的重要增強功能，將在初始推出後新增。
* C — 這些是整合的進一步增強功能，可在「B」要求之後實施。
